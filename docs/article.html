文章的内容结构与效果

—— 简单干净的源码，清爽愉悦的阅读

【摘 要】

一篇文章的内容最多分成五个层级：章/节/区/段/末，每一个层级对应一个片区（section）。每一层通常都有一个标题，起始顶层片区的标题为 <h2>，其余顺次为 <h3> <h4> <h5> <h6>。<h1> 标签用于 <article> 的主标题，通常也是页面的大标题。

<article> 是内容结构的根元素，包含了所有的内容成员。

注： 一篇正常的文章除了主标题外，通常只包含了最多3-4个层级，即：章节区段。再多的层级阅读友好性很差。

内容的结构

下面的逐级嵌套结构按标题号递增，逻辑上是越来越细化的内容区。这是一个常规合理的内容组织形式，它能让文章的内容结构合理，表达清晰，阅读起来也更轻松。

这里的结构主要参考Markdown文档按标题折叠的逻辑来设计，表达合理的同时也便于对Markdown文档的解析结果进行程序处理：重构为如下的层次结构。

注：
根据使用经验，这种规范的内容结构实际上可以迫使您把文章的内容组织得更合理。您可以试一试 :)

层次图

<article> <h1></h1>                             <!-- 根容器：主标题（页面大标题） -->
    <header role="summary"></header>            <!-- 摘要 -->
    <nav role="toc" class="toc"></nav>          <!-- 目录 -->
    <header></header>                           <!-- 导言 -->
    <section> <h2></h2>                         <!-- 章片区/标题。第一层 section -->
        <header></header>                       <!-- 导言 -->
        <section> <h3></h3>                     <!-- 节片区/标题。第二层 section -->
            <header></header>                   <!-- 导言 -->
            <section> <h4></h4>                 <!-- 区片区/标题。第三层 section -->
                <header></header>               <!-- 导言 -->
                <section> <h5></h5>             <!-- 段片区/标题。第四层 section -->
                    <header></header>           <!-- 导言 -->
                    <section> <h6></h6>         <!-- 末片区/标题。第五层 section -->
                        <section role="content" class="content">
                            [块单元内容]...      <!-- 内容/正文 -->
                        </section>
                    </section></section></section></section></section>
    <hr />                                      <!-- 间隔 -->
    <ul role="seealso></ul>                     <!-- 另参见 -->
    <ol role="references></ol>                  <!-- 文献参考 -->
</article>

解析逻辑：

  • 如果内容包含多个层级（标题），在上级标题和首个下级标题之间的内容被视为 <header> 部分。
  • 如果内部已经没有下级标题，则全部视为内容，由一个 <section role="content"> 包含。它被称为 正文区，无导言和标题部分。
  • 如果内部的下级标题不是顺次递增而是跨级了，则解析为内容区容器（内部子标题继续解析）。这违反了标准的层次结构，通常是内容的组织不够合理。
  • 除了终端内容区，一个层级中最多只包含3种子元素：h2-h6、header（前段）和可能多个 section 单元。

注：
对于没有明确标签表达的内容单元，采用相似用途的标签和 role 属性来定义，如 <nav role="toc"> 表示目录单元。
在实际的文章内容里，会添加一个相同名称的 class 属性值，以便于样式定义。如上源码所示。

内容成员

块单元

块单元会占据单独的一行，在CSS中的显示属性是 display:block。块单元又分为两种，一种是需要包含其它元素的容器类，一种是可以直接包含文本内容的内容类。

容器类

内部不能直接包含行内元素（如文本、图片等），如果没有标准的内部元素，则必须采用通用的 <p> 段落容纳。

<header role="summary"></header>    <!-- 摘要：/h3, p... -->
<nav role="toc"></nav>              <!-- 目录：/h4/a, ol/li/a（可多级） -->
<header></header>                   <!-- 导言：/p... -->
<section></section>                 <!-- 片区：/section... -->
<section role="content"></section>  <!-- 正文区：/[^hx, section]... -->
<ul></ul>                           <!-- 无序列表：/li/... -->
<ul role="seealso></ul>             <!-- 另参见：/li/... -->
<ol></ol>                           <!-- 有序列表：/li/... -->
<ol role="cascade"></ol>            <!-- 多级编号列表：/li/ol, ul/li... -->
<ol role="codes></ol>               <!-- 代码表：/li/code/... -->
<ol role="references></ol>          <!-- 文献参考：/li/... -->
<dl></dl>                           <!-- 定义列表：/dt,dd/... -->
<table></table>                     <!-- 表格：/caption, thead, tbody, tfoot/tr/td/... -->
<figure></figure>                   <!-- 插图：/figcaption, p... -->
<blockquote></blockquote>           <!-- 引用块：/p... -->
<aside></aside>                     <!-- 批注：/p... -->
<details></details>                 <!-- 详细内容：/sumary, p... -->
<footer></footer>                   <!-- 尾注：/p... -->
<pre></pre>                         <!-- 代码块：/code/... -->

内容类

<hr />                      <!-- 线条 -->

代码着色

封装规则

考虑简单性，代码内的高亮着色采用 <b> 元素封装，通过 class 类名区分。所有 <b> 元素只会处于同一层级（<code> 直接子元素），没有嵌套。

基本类名定义如下：

b.cmt   /* comments */
b.str   /* string */
b.kws   /* keywords */
b.tag   /* tag-name */
b.slr   /* selector */
b.atn   /* attribute-name */
b.atv   /* attribute-value */
b.fun   /* function-work */
b.typ   /* data-type */
b.imp   /* important */
b.doc   /* <!DOCTYPE...> */
b.cdt   /* <![CDATA[...]]> */
b.rex   /* js-RegExp */
b.non   /* invalid */

<b class="kws">package</b> main

<b class="kws">import</b> (
    <b class="str">"fmt"</b>
    <b class="str">"os"</b>
)

<b class="kws">func</b> main() {
    <b class="kws">var</b> s, sep <b class="typ">string</b>
    <b class="kws">for</b> i := 1; i &lt; <b class="fun">len</b>(os.Args); i++ {
        s += sep + os.Args[i]
        sep = <b class="str">" "</b>
    }

}</code></pre>

效果

// prints its command-line arguments.
package main

import (
    "fmt"
    "os"
)

func main() {
    var s, sep string
    for i := 1; i < len(os.Args); i++ {
        s += sep + os.Args[i]
        sep = " "
    }
    fmt.Println(s)
}
