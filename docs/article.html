文章的内容结构

—— 简单干净的源码，清爽愉悦的阅读

【提 要】

一篇文章的内容最多分成五个层级：章/节/区/段/末，每一个层级对应一个片区（section）。每一层通常都有一个标题，起始顶层片区的标题为 <h2>，其余顺次为 <h3> <h4> <h5> <h6>。<h1> 标签用于 <article> 的主标题，通常也是页面的大标题。

<article> 是内容结构的根元素，包含了所有的内容成员。

注： 一篇正常的文章除了主标题外，通常只包含了最多3-4个层级，即：章节区段。再多的层级阅读友好性很差。

内容的结构

下面的逐级嵌套结构按标题号递增，逻辑上是越来越细化的内容区。这是一个常规合理的内容组织形式，它能让文章的内容结构合理，表达清晰，阅读起来也更轻松。

这里的结构主要参考Markdown文档按标题折叠的逻辑来设计，表达合理的同时也便于对Markdown文档的解析结果进行程序处理：重构为如下的层次结构。

注：
根据使用经验，这种规范的内容结构实际上可以迫使您把文章的内容组织得更合理。您可以试一试 :)

层次图

<h1></h1>                                   <!-- 主标题 -->
<header role="abstract"></header>           <!-- 提要 -->
<nav role="toc"></nav>                      <!-- 目录 -->
<article>                                   <!-- 文章区 -->
    <header></header>                       <!-- 导言 -->
    <h2></h2>                               <!-- 章标题 -->
    <section role="s1">                     <!-- 章片区。第一层 section -->
        <header></header>                   <!-- 章导言 -->
        <h3></h3>                           <!-- 节标题 -->
        <section role="s2">                 <!-- 节片区。第二层 section -->
            <header></header>               <!-- 节导言 -->
            <h4></h4>                       <!-- 区标题 -->
            <section role="s3">             <!-- 区片区。第三层 section -->
                <header></header>           <!-- 区导言 -->
                <h5></h5>                   <!-- 段标题 -->
                <section role="s4">         <!-- 段片区。第四层 section -->
                    <header></header>       <!-- 段导言 -->
                    <h6></h6>               <!-- 末标题 -->
                    <section role="s5">     <!-- 末片区。第五层 section -->
                        <header></header>   <!-- 末导言 -->
                        [ Text ] ...        <!-- 正文内容 -->
                        <footer></footer>   <!-- 末小结 -->
                    </section>
                    ...                     <!-- 平级 h6/section -->
                    <footer></footer>       <!-- 段小结 -->
                </section>
                ...                         <!-- 平级 h5/section -->
                <footer></footer>           <!-- 区小结 -->
            </section>
            ...                             <!-- 平级 h4/section -->
            <footer></footer>               <!-- 节小结 -->
        </section>
        ...                                 <!-- 平级 h3/section -->
        <footer></footer>                   <!-- 章小结 -->
    </section>
    <hr />                                  <!-- 章间隔 -->
    ...                                     <!-- 平级 h2/section -->
    <footer></footer>                       <!-- 结语 -->
</article>
<ul role="seealso"></ul>                    <!-- 另参见 -->
<ol role="reference"></ol>                  <!-- 文献参考 -->

解析逻辑：

  • 每一级标题之后的内容即为该标题相关的 <section>。
  • 如果在内容之中包含层级更低的子标题，首个子标题之前的内容被视为 <header>。
  • 如果内容中已经没有下级标题，则视为正文内容。
  • 如果子标题不是顺次递减而是跨级，也视为正文内容，但内部子标题继续解析。注：这通常是内容组织不太合理的表现。

内容成员

块单元

块单元会占据单独的一行，在CSS中的显示属性是display:block。块单元又分为两种，一种是需要包含其它元素的结构类，一种是可以直接包含文本内容的内容类。

结构类

内部不能直接包含行内元素（如文本、图片等），如果没有标准的内部元素，则必须采用通用的 <p> 段落容纳。

<header role="abstract"></header>   <!-- 提要：Abstract /h3, p... -->
<nav role="toc"></nav>              <!-- 目录：Toc /h4, ol:cascade/li/a（可多级） -->
<ul role="seealso"></ul>            <!-- 另参见：Seealso /li/... -->
<ol role="reference"></ol>          <!-- 文献参考：Reference /li/... -->
<header></header>                   <!-- 导言：Header /h4, p... -->
<footer></footer>                   <!-- 结语/小结：Footer /h4, p... -->

行内单元

只能被包含在内容元素里的元素。即通常所说的内联（inline）元素，其中既有可以包含文本和其它内联元素的内容元素，也有作为内容本身的内容实体元素（如 <img>、<audio> 等）。

行内单元不是独立的内容块，也不是内容件，它们不能单独出现在章节片区（<section>）里，必须被内容行（譬如 <p>）封装才行。

<video></video>                 <!-- 视频：/track,source, #text -->
<picture></picture>             <!-- 图片：/source,img -->

<ruby></ruby>                   <!-- 注音：/rb,rp,rt -->

内容的源码性质与段落类似，但表达了内容的逻辑含义。

预排版

与包含特定语言标注的代码表/代码块不同，这是一种需要保留原始版式的通用结构。

<pre data-tabs="4">
    <!-- 内联单元或文本
        注：
        首个换行会被浏览器友好清理，末尾换行会被原样保留。 -->
</pre>

级联编号表

级联编号表仅由有序列（<ol>）表构成，除了最深那个层级，列表项（<li>）中仅有两个子元素：可选的小标题（<h5>）和子列表（<ol>）。

<ol role="cascade">
    <li></li> <!-- 由CSS表达序号规则 -->
    <li><h5></h5>
        <ol>
            <li></li>
        </ol>
    </li>
</ol>

注：
有序（<ol>）和无序（<ul>）列表的混合嵌套也是一种级联表，但它们属于正常的DOM结构，不在这里特别区分。

代码表

包含 data-lang 属性用于指定代码语言，data-tab 属性指定制表符（Tab）对应的空格数。可以有两级定义：

  1. 根级。书写在 <ol> 元素上，对应整个代码表的默认值。
  2. 行级。书写在 <code> 元素上，对应当前行的语言和制表符宽度（可选）。

元素的 start 属性可以让对整片代码中部分片段的引用变得十分清晰。如示例：源码2（代码表）。

<ol role="codes" data-lang="xx" data-tab="4" start="x">
    <li><code> <!-- 根封装 -->
        ...<b></b>... <!-- 代码内容 -->
    </code></li>
    <li><code data-lang="xx" data-tab="2"> <!-- data-xx定义覆盖 -->
        ...
    </code></li>
</ol>

代码块

与代码表相似，包含 data-lang 和 data-tab 两个属性定义代码的特性，书写在 <code> 元素上。

<pre> <!-- 容许简单换行 -->
<code data-lang="xx" data-tab="4"> ...<b>..</b>...
<!-- Tab符被自动转换为空格（如果配值非零） -->
</code><pre> <!-- 末尾应当紧跟 -->

空白块

这是一种特别的内容单元，主要用于某种交互展示。在传统纸质的内容中没有这样的逻辑，但在计算机的动态交互文档中应当是一种必要的概念补充。

<div role="space">
    <!-- 内联单元或文本 -->
</div>

注解

用于解释正文中具体的名词或词组。

<p role="note">
    <!-- 内联单元或文本 -->
</p>

注：
与批注块（<aside>）单元相比，注解 逻辑上是直接与目标相关联的。

代码着色

封装规则

代码内的各种单元采用 <b> 元素封装，由 role 定义单元类型，但注释类型采用 <i> 元素封装。通常所有的 <b> 和 <i> 元素处于同一层级（<code> 的直接子元素），没有嵌套。

基本名称（简写）定义如下：

简写      // 选择器        说明
-----------------------------------------------------------
kw        // b[role=kw]   keyword 关键字
str       // b[role=str]  string 字符串
fn        // b[role=fn]   function-name 函数名
val       // b[role=val]  value 通用值类型
dt        // b[role=dt]   data-type 数据类型
tag       // b[role=tag]  tag-name HTML标签名
an        // b[role=an]   attribute-name 属性名
imp       // b[role=imp]  important 重要
doc       // b[role=doc]  <!DOCTYPE...> HTML文档声明
cd        // b[role=cd]   <![CDATA[...]]> HTML代码片段
re        // b[role=re]   RegExp 正则表达式（如：/.../gi）
err       // b[role=err]  error 错误提示

注：
用 <b> 封装着色文本可减少书写，同时在没有样式渲染时，默认的粗体显示也较友好。
用 <i> 封装注释类型效果类似，同时如果注释内还包含更多标记时，结构也较为清晰。

效果

// prints its command-line arguments.
package main
import (
    "fmt"
    "os"
)
func main() {
    var s, sep string
    for i := 1; i < len(os.Args); i++ {
        s += sep + os.Args[i]
        sep = " "
    }
    fmt.Println(s)
}
